what we do

Here's what we do.

As a data scientist/analyst (detective), we source for data for projects, validate the data, clean or wrangle it, visualize it using python packages and R, in order to draw actionable insights. we also build prective models using selected features to make future predictions.

Data collection and preprocessing

Here, we focus on gathering relevant data from various sources. This might involve web scraping, working with APIs, accessing databases, or using other data acquisition methods. Once the data is gathered, we preprocess and clean it to ensure it's in a suitable format for analysis. This may include handling missing values, data transformation, and dealing with outliers. All of these steps depend on the nature of the data in question

Data Analysis and Modeling

In this category, we perform exploratory data analysis (EDA) to understand the patterns, relationships, and insights within the data. Visualization tools and techniques can be utilized to help with this process. Once we have a good grasp of the data, we'll apply various machine learning algorithms and statistical models to build predictive or descriptive models. This could involve tasks such as regression, classification, clustering, and more. Evaluating model performance and fine-tuning the models also fall under this category.

Interpretation and Communication

This involves interpreting the results of the data analysis. We communicate our findings effectively to stakeholders. We create data visualizations, reports, and presentations to convey complex insights in a clear and understandable manner. The goal is to provide actionable recommendations and insights that can guide (business) decision-making within the organization or project.

See some of my recent work

Here are some projects I worked on recently.

Loan default prediction model

This was a team project involing designers, front-end and back-end developers and data scientist. The data science team comprising of three team members in which I was the team lead, gathered data using a python package called Faker, validated it using charts, graphs etc, cleaned and visualized it then did feature selection and built a predictive model using Gradient boosting classifier with an accuracy of 79%. We saved the model using picke package, used it to build an API using FastAPI, deployed it on Railway and handed it over to the back-end developers for integration into the application.

Flights database insights

In this project, we queried the "Flights" database to gain actionable insights into the database

IBM employee attrition

Employee attrition measures how many workers have left an organization and is a common metric companies use to assess their performance. There are often many reasons why employees leave an organization. In this project, we explored a dataset created by IBM data scientists to analyze the factors that lead to employee attrition. The data consists of about 1470 current and former employees with information related to their Monthly salary, job satisfaction, work life balance, experience, distance from home etc. We carried out exploratory data analysis based on some questions.
